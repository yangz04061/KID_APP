# 幼儿英语启蒙 App 信息架构与页面流程（v1）

## 1. 文档目标

本文档基于已确认需求，输出首期产品的信息架构、页面层级和核心流程，用于后续低保真原型与技术设计。

## 2. 设计前提

- 平台：网页端优先，重点适配 iPad。
- 使用方式：家长陪同使用为主。
- 内容形式：偏小游戏探索。
- 语言策略：英文单词 + 英文发音 + 中文释义。
- 首期主题：动物、汽车、工程车。

## 3. 信息架构

```text
App 入口
├─ 欢迎页 / 启动页
├─ 首页 / 主题选择
│  ├─ 动物主题
│  ├─ 汽车主题
│  ├─ 工程车主题
│  └─ 家长入口
├─ 主题页
│  ├─ 主题引导
│  ├─ 单词预览
│  ├─ 小游戏 1：听声音选图片
│  ├─ 小游戏 2：图片配对
│  ├─ 小游戏 3：找一找 / 点一点
│  └─ 主题完成页
└─ 家长区
   ├─ 学习进度
   ├─ 使用设置
   └─ 产品说明
```

## 4. 页面说明

### 4.1 欢迎页 / 启动页

作用：

- 快速进入产品。
- 建立轻松、友好的第一印象。

页面元素：

- 品牌名或产品名。
- 1 个明显的“开始”按钮。
- 简单动态插画，例如动物或工程车轻微摆动。

设计要求：

- 停留时间短，可 1 次点击进入首页。
- 不堆叠信息，不出现复杂引导文案。

### 4.2 首页 / 主题选择

作用：

- 让孩子快速选择感兴趣的主题。
- 让家长快速理解产品结构。

页面元素：

- 3 个主题卡片：动物、汽车、工程车。
- 角落的家长入口。
- 可选的最近学习记录提示。

设计要求：

- 主题卡片足够大，适合低龄儿童点击。
- 每个主题卡片使用单独插画和颜色。
- 页面上不出现多层导航。

### 4.3 主题页

作用：

- 承载一个主题内的完整学习闭环。
- 让孩子从“看到主题”自然过渡到“开始探索和游戏”。

页面结构建议：

- 顶部：主题名称与返回首页按钮。
- 中部：主题场景图或主题主视觉。
- 下部：开始探索按钮。

设计要求：

- 主题页要更像进入一个“小世界”，而不是课表页面。
- 动物主题可用动物园或草地场景。
- 汽车主题可用道路或城市场景。
- 工程车主题可用工地场景。

### 4.4 单词预览页

作用：

- 在进入游戏前，用最短路径让孩子先建立基础认知。

页面元素：

- 大图片。
- 英文单词。
- 中文释义。
- 发音按钮。
- 下一张按钮或自动播放。

设计要求：

- 一次只展示 1 个单词对象。
- 图片优先级最高，英文单词次之，中文释义再次之。
- 总时长短，建议每个主题预览 3 至 6 张核心卡片即可进入游戏。

### 4.5 小游戏页

作用：

- 通过互动强化识别和记忆。

共性元素：

- 当前题目的语音提示。
- 可点击或拖拽的目标对象。
- 即时反馈动画。
- 进度提示。

共性要求：

- 单局规则稳定，不在中途切换玩法。
- 单局控制在 1 分钟以内。
- 支持重试和快速进入下一题。

#### 小游戏 1：听声音选图片

流程：

- 播放单词发音。
- 显示 2 至 3 个候选图片。
- 孩子点击正确图片。
- 立即给出鼓励反馈。

#### 小游戏 2：图片配对

流程：

- 显示图片项和目标槽位。
- 孩子拖拽或点击配对。
- 配对成功后出现吸附或点亮反馈。

#### 小游戏 3：找一找 / 点一点

流程：

- 显示场景图。
- 通过发音或简短文字提示目标。
- 孩子在场景中找到目标并点击。
- 成功后播放动画并进入下一题。

### 4.6 主题完成页

作用：

- 给孩子明确完成感。
- 给家长简单反馈孩子本轮成果。

页面元素：

- 星星、贴纸或徽章。
- 本次完成的单词数。
- 再玩一次按钮。
- 返回首页按钮。

设计要求：

- 奖励明确但不过度刺激。
- 再玩入口足够突出，鼓励重复练习。

### 4.7 家长区

作用：

- 让家长查看学习情况并调整设置。

页面结构：

- 学习进度。
- 使用设置。
- 产品说明。

设计要求：

- 不让孩子轻易误入。
- 信息表达清晰，不使用复杂专业术语。

## 5. 核心用户流程

### 5.1 主流程：家长陪同开始一次学习

```text
打开应用
→ 进入欢迎页
→ 点击开始
→ 进入首页
→ 选择主题
→ 进入主题页
→ 浏览单词预览
→ 进入小游戏
→ 完成主题奖励
→ 返回首页或重复挑战
```

### 5.2 家长查看进度流程

```text
首页
→ 进入家长区
→ 查看学习进度
→ 调整设置
→ 返回首页
```

### 5.3 重复练习流程

```text
主题完成页
→ 点击再玩一次
→ 返回当前主题小游戏
→ 完成下一轮挑战
```

## 6. 页面跳转关系

- 欢迎页可进入首页。
- 首页可进入任一主题页。
- 首页可进入家长区。
- 主题页进入单词预览页。
- 单词预览页进入小游戏流程。
- 小游戏完成后进入主题完成页。
- 主题完成页可返回首页或重新开始当前主题。
- 家长区可返回首页。

## 7. MVP 页面清单

- 欢迎页。
- 首页 / 主题选择页。
- 主题页。
- 单词预览页。
- 听声音选图片页。
- 图片配对页。
- 找一找 / 点一点页。
- 主题完成页。
- 家长区进度页。
- 家长区设置页。

## 8. 原型阶段重点

- 优先验证首页到主题页的进入是否足够自然。
- 优先验证“单词预览 + 小游戏”切换是否顺滑。
- 优先验证场景化小游戏是否比卡片式页面更能吸引孩子。
- 优先验证中文释义是否会干扰儿童主界面视觉焦点。
- 优先验证家长入口是否足够隐蔽但不难找。
